## 1. 产品定位的重新锚定

**原定位**：learn-mate 是一个"教 Harness Engineering 的 CLI"——这是 v0.1.0 发布时的思路。

**新定位**：learn-mate 是**课程交付基建**。真正的商品是**课程内容**，CLI 只是载体。

### 为什么转向

"Java + harness engineering" 作为副业变现路径不是好选择：

- **受众窄**：Java 开发者 ∩ 对 AI agent 感兴趣 ∩ 能接受中文文档 ∩ 愿意花 API 费用试错——交集小
- **装机门槛高**：要装 Node + Java + Maven + 配 LLM key，非技术背景直接劝退
- **付费意愿低**：技术人喜欢"自己搭不花钱"，且这门课内容偏深，不在"焦虑购买"心智里
- **同质化竞争**：和 Claude Code / Cursor 自己的 docs 直接抢注意力，劣势明显

关键变量是**用户手上有 LLM 入门资料存量**——这推开了另一扇门。

## 2. 新锚点：LLM 入门课程

### 为什么 LLM 入门是对的选择

| 维度 | Harness Engineering（原主打） | LLM 入门（新主打） |
|---|---|---|
| 潜在学员基数 | 小（Java + agent 开发者交集） | 大（所有想入门的人） |
| 焦虑感 / 付费意愿 | 中（已会 AI，只想精修） | **高**（焦虑、信息过载） |
| 需要编程背景 | 是（Java + Maven） | 否 |
| 市场同质化程度 | 低（有独特观察） | 高（视频课泛滥） |
| **差异化切入点** | 技术深度 | **"AI 带学 + 强制验收，不是视频"** |

LLM 入门市场已有大量视频课，但**几乎没有"带学 + 强制验收"形态**——learn-mate 的 `judge prompt + structured output` 机制是天然差异化。视频课用户听着听着睡着也能"过"；learn-mate 的课让 judge 判不过就过不了——这种"**焦虑型学员也许会喜欢的反模式**"是可以打的差异点。

### 产品线重整

```
免费层（引流）
  ├─ LLM 入门 trial（前 2-3 step）
  └─ 内容营销：即刻 / 小红书 / V2EX / X thread
     │
     ↓ 转化
付费层（主收入）
  ├─ LLM 入门完整版         ¥99-199
  ├─ LLM 应用开发实战       ¥299-499
  └─ Harness Engineering    ¥499-999   ← 现在这个 MVP，作为系列最高阶
     │
     ↓ 溢价
咨询层（拉高收入上限）
  ├─ 一对一教练  ¥500-1000/小时
  └─ 团队培训    ¥5000-15000/场
```

**Harness Engineering 的新角色**：从"主打"变成"系列末端的高阶课"。不是降级，是重新定位——它是"你已经会用 agent 了，想精修 harness"的目标人群用的，**不该**承担拉新任务。

## 3. 三个月分阶段路线图

### 第 1 个月：验证 learn-mate 能装下 LLM 入门

**核心问题**：你的 LLM 入门资料能不能塞进 learn-mate 框架？

具体动作：
1. 把 LLM 入门资料拆成 **5-7 个 step**（每步 15-30 分钟能做完）
2. 做新课程 `data/llm-intro.json` + `data/llm-intro/step-*.md`
3. **关键决策**：砍掉 Java 依赖——LLM 入门这门课不该强制装 Maven
4. **真的找 3-5 个非程序员朋友跑一遍**——第 1 个月唯一真正重要的事

**成功标准**：3 人能跑完全部 step + **至少 2 人说"这种学法和视频课不一样"**

### 第 2 个月：做减法 + 商业化基建

1. 按第 1 个月反馈砍 step、调难度、重写 judge prompt
2. 分层：免费版（trial） + 付费版（完整 + 答疑）
3. 一键安装脚本 `curl ... | bash`，隐藏技术细节
4. 选分发平台：**小报童 / 知识星球 / 即刻**（**不要自建网站**）
5. 写 3-5 篇内容营销素材

**成功标准**：课程目录定稿 + 分发平台选定 + 内容营销素材就位

### 第 3 个月：上线 + 前 10 个付费学员

1. 定价 **¥99-199** 之间第一期友情价
2. 每周 1 条 "学员故事" / "常见错误" 内容
3. 目标：**10 个付费学员**（不是 100）
4. 每个付费学员**单独问**：哪里卡住、愿意为什么进阶内容付更多钱

**成功标准**：10 个付费学员 + 他们的反馈文档

## 4. learn-mate 承接 LLM 入门的最小改造

总工作量 **~1 周**，比继续啃 BACKLOG 价值高得多：

| 改动 | 原因 | 工作量 |
|---|---|---|
| 课程可以只有讲义 + llmChecks，不强制 rules | LLM 入门没有"跑测试"类客观指标 | 0.5d |
| `init` 按 `course.requires` 决定装什么 | 不应让 LLM 入门学员装 Maven | 2-3d |
| 支持 `template: none`（纯阅读 + 答题） | 见上 | 2d |
| 一键安装脚本 `curl ... \| bash` | 非程序员装 npm 都心虚 | 0.5d |

**铁律**：**在做这批改造之前，不要动 BACKLOG 的任何一条。** 先让 LLM 入门跑起来，再回头修 harness engineering 的小毛病。

---

## 5. 要避开的坑（这些会让你 6 个月后一无所获）

1. **"再加一个功能就完美了"循环** — BACKLOG 11 条全做完也不会变出用户
2. **等完美才上线** — 第一期就是 MVP，有反馈比完美重要
3. **同时做多门课** — 先把 LLM 入门做到真的有人付费，再考虑第二门
4. **免费策略过度** — 永远免费 = 永远没钱，哪怕 ¥9.9 也要有付费门槛才能筛出真学员
5. **自建 SaaS** — 服务器费 + 运维成本会吃掉副业 ROI
6. **单独卖 learn-mate 工具** — 技术人不会为工具付费，他们会自己搭

## 6. 可行性分级（不同"成功"标准分别判）

| 成功定义 | 判断 | 关键要素 |
|---|---|---|
| 副业 **¥2000-5000/月**，做自己喜欢的事 | **可能** | 12 个月认真投入 + LLM 入门是主战场 |
| 副业 **¥10000+/月**，半职替代 | **很难但不是不可能** | 2-3 年持续经营 + 社群声誉 |
| learn-mate repo **1000+ star / 外部贡献者** | **低概率，且不是真正该追的** | 你追的是学员付费，不是 star |
| 个人作品集 / 博客素材 / 面试谈资 | **已经成功** | 不用加功能，现在就能写文章 |
| 被 Anthropic 生态注意到（DevRel / reference） | **被低估的一条** | 成本：翻译 real-qwen-validation.md 为英文 + 一条 X thread |

**最被低估 / 成本最低的一条**：把 `docs/real-qwen-validation.md` 翻译为英文，配图发一条 X thread，at Anthropic 相关开发者账号。成本 1-2 天，上限可能很高。

## 7. 现实预期（别骗自己）

- **前 6 个月收入预期：¥0-3000/月**。前期是**亏时间**——写内容 + 答疑 + 打磨产品
- **¥5000/月稳定副业**：至少 **12 个月**累积
- **别承诺"包学会"**——这是灾难起点
- **别期待"发一篇文章就爆"**——内容平台的量级是**复利级**，不是爆发式
