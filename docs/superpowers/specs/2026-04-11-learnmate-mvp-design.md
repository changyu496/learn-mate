# LearnMate MVP 设计文档 (v2)

> **核心定位**：不是课程执行器，而是一个懂你的 AI 私教

## 产品愿景

LearnMate 不是把课程内容塞给用户，而是让 Agent **理解课程后，用最适合你的方式教你**。

想象：5年后，LearnMate 是一个虚拟人，能用文字、语音、视频跟你互动，了解你，记住你，陪你成长。

**今天的 MVP**：文字版私教，验证学习模式是否有效。

---

## 核心原则

1. **掌握式学习** — 做对才算会，不跳过验证
2. **苏格拉底式教学** — 通过提问引导你自己思考
3. **个性化** — 根据你的水平、偏好、习惯调整讲解
4. **记忆** — 记住你，陪你成长，越学越懂你

---

## 验证方式设计（基于教育理论）

### 理论基础

**1. 测试效应 (Testing Effect)**
- 研究发现：自我测试比反复阅读更有效
- 原理：提取记忆的过程会强化记忆

**2. 生成效应 (Generation Effect)**
- 自己生成的答案比被动接收记得更牢
- 让学生自己解释概念，比看解释学得深

**3. 布鲁姆分类法 (Bloom's Taxonomy)**
```
记忆 → 理解 → 应用 → 分析 → 评估 → 创造
```
越往上代表理解越深

**4. 形成性评估 vs 总结性评估**
- 形成性：学习过程中的检查，用来调整教学
- 总结性：学完后考试

### 三层验证体系

**第一层：理解检查（必做）**
每次讲完一个概念，让学生用自己的话解释：
```
私教：你刚才学到了 Harness 是"控制 LLM 的一切装备"。
       你能用自己话说说这是什么意思吗？
```
Agent 判断是否抓住了核心概念。

**第二层：应用检查（核心）**
问一个延伸问题，让学生应用：
```
私教：好，现在你理解了 Harness 的概念。
       如果让你设计一个 Agent 的 Harness，你会包含哪些部分？
```
Agent 判断是否能迁移应用。

**第三层：微练习（可选）**
给一个小任务，让用户生成：
```
私教：来个小练习——帮我写一段 prompt，
       让 LLM 扮演一个有工具的 Agent
```
Agent 判断是否能实际操作。

### 动态调整策略

根据用户表现，动态调整节奏：

| 用户表现 | Agent 策略 |
|---------|-----------|
| 回答流畅 | 加快节奏，可以一次讲多个概念 |
| 回答迟疑 | 放慢节奏，多给例子 |
| 答错了 | 换角度讲，从不同方式解释 |
| 答对了 | 表扬 + 继续深入 |

## 学习流程设计

### 旧流程（课程执行器）
```
显示课程内容 → 出选择题 → 用户选 → 对/错
```

### 新流程（AI 私教）
```
了解用户 → Agent 消化课程 → 对话式讲解 → 验证理解 → 答错继续教 → 下一知识点
```

### 具体流程

```
1. 入课对话
   Agent 通过对话了解用户背景、目标、偏好
   → 建立用户画像，存入记忆

2. 知识点学习（循环）
   a) Agent 消化课程内容，用自己的话讲出来
   b) 根据用户水平调整讲解深度（分步讲解，不一次性给太多）
   c) 用提问代替陈述，引导用户思考
   d) 验证理解：出题或让用户复述
   e) 如果答错：从不同角度再讲，直到真正掌握
   f) 进入下一知识点

3. 持续记忆
   - 记住用户学过的、答错的、薄弱点
   - 下次学习自动从薄弱点复习
   - 持续观察用户习惯，优化讲解方式
```

## 课程内容 vs 私教能力

| 课程内容（输入） | 私教能力（价值） |
|-------------|-------------|
| Harness 是什么 | Agent 理解后，用你能懂的话讲 |
| 12 个 Lectures | 对话式学习，不是阅读 |
| 多语言代码示例 | 记住你偏好 Java，用 Java 举例 |
| 练习题 | 不暴露答案，让你自己做 |

### 对话式学习示例

**入课对话：**
```
私教：你之前有 Agent 开发经验吗？比如 LangChain、AutoGPT 这类的？
用户：用过一点 LangChain，但感觉挺迷糊的
私教：好，那我知道了。我会从基础开始讲。Harness 其实就是...
```

**讲解（分步）：**
```
私教：Harness 这个词你可能觉得陌生，但它就像骑马时的装备。
      你骑马的时候，缰绳是控制马的，马鞍是让你坐稳的，马是本身的能力。
      LLM Agent 也一样，模型是"马"，Harness 是那些让你控制它、让它做事的"装备"。
用户：哦！那 Harness 具体包括哪些东西？
私教：好问题！主要包括三样：...
```

**验证理解：**
```
私教：讲完了，我问你一个问题：如果 LLM 很强，但 Harness 设计得不好，会发生什么？
用户：可能会控制不好它，让它做错事？
私教：太对了！就像给了一匹千里马，但没有好的马鞍和缰绳，你也骑不好...
```

## 用户记忆设计

### 用户画像
```typescript
interface UserProfile {
  id: string;
  name: string;
  level: 'beginner' | 'intermediate' | 'advanced';
  preferredLanguage: 'python' | 'java' | 'javascript' | 'go';
  learningStyle: 'slow' | 'fast' | 'deep' | 'practical';
  interests: string[];
  goals: string;
  createdAt: Date;
}
```

### 学习记忆
```typescript
interface LearningMemory {
  lectureId: string;
  status: 'not_started' | 'in_progress' | 'mastered';
  attempts: number;
  weakPoints: string[];      // 薄弱点
 讲解偏好: string[];        // 用户喜欢怎么讲
  lastReviewed: Date;
}
```

### 对话记忆
```typescript
interface ConversationMemory {
  userId: string;
  lectureId: string;
  messages: Message[];       // 对话历史
  lastTopic: string;        // 上次讲到哪
}
```

## CLI 命令（简化）

```bash
learn-mate start     # 开始学习（进入对话）
learn-mate continue  # 继续学习
learn-mate progress  # 查看进度
learn-mate reset     # 重置进度
```

**不需要的命令**（因为是对话式）：
- `plan set` → 对话中完成
- `answer` → 对话中完成

---

## 成功标准

- 用户感觉在跟一个真实的私教对话，而不是在读文档
- Agent 能记住用户，第二次对话时主动问"上次你说到 XX，感觉怎么样？"
- 用户答错时，Agent 换种方式讲，直到用户真正理解
